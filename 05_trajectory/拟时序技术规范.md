## Trajectory Purpose

Trajectory analysis 的目的不是单独证明某个 subtype 必然转化为另一个 subtype。

本章目的：

```text
在已确定的 malignant subtype framework 下，检验 4 个恶性亚型是否沿连续转录状态排列，
并评估该 ordering 是否与 proliferation / stemness / Neftel state / NMF metaprogram 变化一致。
```

审稿风险边界：

- 不写 deterministic transition，除非后续有更强证据。
- 优先写 inferred continuum、trajectory-compatible ordering、pseudotime-associated gradient。
- root 不能只由 GAP43 决定，需要 stemness、SOX family、cell cycle、Neftel state、NMF program 和 graph structure 共同支持。

## Chapter-Level Scientific Position

Trajectory analysis 不重复 chapter 5 的任务。Chapter 5 已经回答 malignant cells 是否能稳定分成 4 个 transcriptional subtypes；本章补的是 state progression dimension。

本章围绕 4 个问题组织：

1. 连续性：NPC-P、OPC-M、MES-V、MES-I 是彼此孤立的 clusters，还是同一张 transcriptional state landscape 上的区域。
2. Hierarchy：哪个 subtype 最 plastic，是否支持 NPC-P 作为 root candidate。
3. MES 多样性：MES-V 与 MES-I 是同一 MES continuum 上的不同位置，还是从 NPC-P 出发的两个 branch。
4. 驱动机制：沿 pseudotime 或 branch 分化出现哪些 switch genes / driver genes，并评估其 biological 和 clinical relevance。

Manuscript narrative：

```text
Subtype analysis defines malignant transcriptional identities.
Trajectory analysis asks whether these identities are embedded in a continuous
state landscape and identifies lineage-compatible branch structure and switch genes.
```

Reviewer-facing alignment：

```text
R1 motivation / insight:
trajectory adds state progression, branch structure, and switch genes beyond subtype discovery.

R2 root / GAP43 concern:
root is justified by multi-metric convergence and then tested by sensitivity analysis;
GAP43 is downgraded from sole root marker to an audited but non-decisive marker.
```

## Chapter Roadmap

Phase 0 — Root justification：

```text
Tools: CytoTRACE2, cell cycle, SOX family, Neftel AMS.
Question: which subtype is eligible as primary root?
Current decision: NPC-P locked as primary root candidate.
```

Phase 1 — Primary trajectory：

```text
Tools: Slingshot primary trajectory + PAGA or kNN-based connectivity sanity check.
Inputs: malignant-only UMAP, four subtype labels, NPC-P root.
Question: how many lineages start from NPC-P, and which subtype is each lineage-compatible endpoint?
Main outputs: per-cell pseudotime, lineage assignment, UMAP lineage curves, 4 x 4 connectivity graph.
```

Phase 2 — Robustness / sensitivity：

```text
Tools: Monocle3 as independent trajectory method; alternative roots such as OPC-M / MES-V / algorithm-defined root.
Question: are lineage ordering and endpoint relationships stable to method and root choice?
Main outputs: method-consistency and root-stability evidence.
```

Phase 3 — Biology overlay：

```text
Tools: NMF MP01-MP06, Neftel state scores, proliferation score, CytoTRACE2 over pseudotime.
Question: does pseudotime track interpretable biological gradients rather than numerical embedding artifacts?
Main outputs: smoothed biological curves along pseudotime / lineage.
```

Phase 4 — Switch genes / driver genes：

```text
Tools: tradeSeq GAM or Monocle3 graph_test.
Question: which genes change along pseudotime or distinguish branches?
Gene classes: early genes, late genes, branch-specific switch genes.
Main outputs: top switch gene table, gene x pseudotime heatmap, lineage-specific GO / Reactome enrichment.
```

Phase 5 — Clinical / external validation：

```text
Tools: TCGA / CGGA bulk RNA-seq projection, survival analysis, optional spatial mapping if available.
Question: do trajectory endpoint or switch-gene signatures carry clinical relevance?
Main outputs: prognostic signature score and survival association.
```

Phase 6 — Manuscript / response integration：

```text
Goal: integrate final trajectory figures, source data, caveats, and response-letter logic.
Core wording: inferred trajectory / lineage-compatible ordering / transcriptional gradient.
Avoid wording: transition into / transform into / differentiate into / evolves into.
```
