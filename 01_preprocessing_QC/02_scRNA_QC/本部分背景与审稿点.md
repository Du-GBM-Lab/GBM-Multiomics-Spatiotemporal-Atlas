# scRNA-seq QC / doublet / malignant identification 背景

## 1. 为什么先做这一部分

这一部分直接回应两个 reviewer 对单细胞基础流程的质疑。

Reviewer 1 指出原稿保留 `<40% mitochondrial content` 的细胞过于宽松，并要求重新检查 high mitochondrial cells、doublet removal 等 QC 指标。Reviewer 2 指出原稿用 EGFR、SOX2、OLIG2、GFAP 等 lineage markers 定义 malignant cells 不严谨，因为这些基因不是 tumor-specific markers；同时旧 CNV 证据弱且不一致。

因此本部分目标不是重做所有下游分析，而是先把最基础的输入细胞集重新定义清楚：哪些细胞通过 QC，哪些是 doublet，哪些细胞由 CNV 证据支持为 malignant-like。

## 2. 当前对象边界

输入对象是：

```text
重新分析\GBM.RNA.integrated.24.rds
```

对象检查显示它是 Seurat object，包含 149,048 cells、24 个样本，样本字段为 `Pt_number`。虽然文件名包含 `integrated`，但 `RNA` assay 保留了 raw-count-like UMI counts，因此可以从 `RNA@counts` 重新进行 QC、normalization、DoubletFinder 和 inferCNV。

需要在方法中避免写成“完全从 10x raw matrix 开始”，更准确的表述是：基于 integrated Seurat object 中保留的 RNA raw counts 重新分析。

## 3. QC 策略

QC 采用两层规则：

1. `percent.mt < 20%` 作为硬阈值，直接回应 reviewer 对原稿 `<40%` 过宽的质疑。
2. 在每个 `Pt_number` 样本内做 MAD-based filtering，用于识别 `nCount_RNA`、`nFeature_RNA`、`percent.mt` 的样本内离群细胞。

MAD 是 median absolute deviation，即围绕中位数的稳健离散度。它比 mean / standard deviation 更不容易被极端细胞影响，适合样本间测序深度差异较大的 single-cell QC。

## 4. Doublet 策略

Doublet 使用 DoubletFinder，并按 `Pt_number` 单样本分别运行。

doublet 是两个细胞进入同一个 droplet / barcode 后被测成一个细胞。它发生在单个 10x library 或单个样本内部，所以不应该把 24 个样本合并后整体估计。脚本中每个样本会单独 normalization、HVG、PCA、neighbors、clustering、pK selection 和 DoubletFinder calling。

## 5. Malignant identification 策略

malignant identification 以 sample-wise inferCNV 为主。

每个样本内单独运行 inferCNV，reference group 使用该样本内较可信的 non-malignant cells，例如 immune cells 和 vascular cells。这样可以避免跨样本整体 inferCNV 带来的 patient effect / batch effect 混杂。

EGFR、SOX2、OLIG2、GFAP 等基因不再作为 malignant classification criterion，只在 inferCNV 之后用于 marker validation 和生物学解释。

## 6. 本部分预期输出

QC / DoubletFinder 输出：

- sample-wise MAD QC thresholds
- filtering 前后每样本细胞数
- QC metric distributions before / after filtering
- DoubletFinder per-cell calls
- 每样本 doublet 比例
- QC 后 cell type composition
- QC + doublet 后 filtered Seurat object

inferCNV 输出：

- sample-wise inferCNV manifest
- per-cell CNV burden score
- malignant-like / non-malignant-like CNV calls
- 每样本 malignant / non-malignant composition
- marker validation plot

## 8. 代码 review 后修正

已根据外部 review 修正以下问题：

1. 三个脚本入口均加入 `JoinLayers()` 兜底，避免 Seurat v5 split layers 影响 `PercentageFeatureSet()`、`NormalizeData()`、`GetAssayData()`。
2. 输出大 Seurat 对象从 `saveRDS()` 改为 `qs2::qs_save()`，符合 `technical-defaults` 中 R object persistence 默认。
3. `nCount_RNA` 和 `nFeature_RNA` 的 MAD filtering 改为在 `log1p()` scale 上计算，阈值表同时保存 log scale 和原始 scale。
4. DoubletFinder 每个样本入口设置 sample-specific seed，增强可重复性。
5. DoubletFinder `paramSweep()` 加入 try-catch，失败时回退到 `default_pk`。
6. 小样本跳过 DoubletFinder 时标记为 `Skipped_low_cells`，不再默默写成 `Singlet`。
7. `qc_pass_final` 显式处理 `NA`，避免依赖隐式过滤。
8. inferCNV 主循环加入 try-catch，单个样本失败不会中断全部样本。
9. inferCNV 默认使用包内 `human_gencode.v40.genes.bed` gene order file，不再强制外部占位路径。
10. inferCNV summary 中 CNV burden 改为 squared deviation：`colMeans((expr - 1)^2)`。
11. inferCNV dense matrix 不再强转 sparse，避免内存反优化。
12. marker validation 前加入 `NormalizeData()` 兜底，并加入 `MKI67`。
